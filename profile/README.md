## 🌈 Contribution Guidelines

Welcome to the Legal Hackathon 2026 project (currently called **GameLL or gamell** short for Gamified Law Learning 👀).  
We’re building something meaningful—please keep contributions thoughtful, clear, and collaborative.

---

### 🧭 Getting Started
- Fork the repository and clone it locally  
- Create a new branch before making changes  
  - Naming examples:
    - `feature/quiz-engine`
    - `fix/login-bug`
    - `docs/update-readme`
- Ensure the project runs correctly before making changes  

---

### 🛠️ How to Contribute

#### 1. Pick or Create an Issue
- Check existing issues first  
- If your idea doesn’t exist, open a new issue and include:
  - What you want to add/change  
  - Why it matters  
  - Suggested approach (optional but helpful)  

#### 2. Work on Your Changes
- Keep changes small and focused  
- Follow the existing structure and coding style  
- Write clear, readable code  
- Add comments for complex logic where needed  

#### 3. Commit Properly
Use meaningful commit messages:
- `feat: add onboarding quiz flow`
- `fix: resolve authentication error`
- `docs: update contribution guide`

---

### 🔍 Pull Request Guidelines
When submitting a PR:
- Link the related issue (e.g. `Closes #12`)  
- Clearly explain:
  - What you changed  
  - Why you changed it  
- Add screenshots for UI changes (if applicable)  
- Make sure your code runs without errors  
- Be open to feedback and revisions  

---

### 🧪 Code Quality & Standards
- Keep code clean and consistent  
- Avoid unnecessary dependencies  
- Test your changes before submitting  
- If applicable:
  - Add or update tests  
  - Ensure existing features are not broken  

---

### 🎨 Frontend Contributions
- Maintain consistent UI/UX  
- Ensure responsiveness across devices  
- Keep components modular and reusable  
- Consider accessibility (color contrast, readability, etc.)  

---

### ⚙️ Backend Contributions
- Follow API structure and naming conventions  
- Validate inputs and handle errors properly  
- Write clean, maintainable logic  
- Keep security in mind (important for legal-related features)  

---

### 💬 Communication
- Be respectful and constructive  
- Ask questions early if unsure  
- Clarify instead of assuming  

---

### 🚫 What to Avoid
- Large, unrelated changes in a single PR  
- Breaking existing functionality without discussion  
- Making changes without explanation  

---

### 🚀 Final Note
We’re building **GameLL/gamell** to help everyday Nigerians better understand their rights and make informed decisions.

Every contribution should support that goal.
If it doesn’t improve clarity, usability, or impact—rethink it.
